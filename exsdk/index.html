exSDK


Introduction

exSDK is a 2D game engine I am developing now. The engine is written in C, and embeds a Lua script intepretor. It aims to help developers fast prototyping 2D games on mobile platform.

exSDK is still in a very early stage. The engine is inspired by Entity-Component system. The script interface is totally learn from Unity3D, and I am currently design a script-driven GUI framework for both Game and Editor.

The user will writting game logics in Lua script. A Lua console and a Lua debugger will be embeded directly in the editor. The editor will include several tools for making 2D resources easily. The goal of exSDK is become small, fast and cross-platform.

My first approach

At the very beginning of the project, I use my own Open-Source C++ framework — exlibs and built the Editor in .NET framework.
I was using SWIG to expose C++ classes to C# Editor, and I choose Python as my game engine script language.

The second time

The C# prototype proofs the Script Driven Entity-Component system is feasible. Since I use Python as my script language, I had to expose C++ interface to Python and C# at the same time. I use BoostPython to ease the job from C++ to Ptyhon. For C# it was done by SWIG. This means if a C++ class changes, I have to modify the Python, BoostPython, SWIG and C# interface either. I smelled the bad code and started thinking, if I choose C/C++ GUI framework, I don’t worry about the interface exposed to C#.

Then I choosed Qt Framework. The Qt shows some promiss and it is really powerful to creating basic GUI through the Signal-Slot architecture. I use it build an Editor which can editing Python script in the runtime. I also make it possible to showing the scripts’ member data in the Properties Window dynamically.
Even more, The engine can detect changes of the Resources such as Texture and Font editing from external tools and reimport them automatically. This prototype can be downloaded below:

A brand new version

When I started porting my code to Mobile Platform (Android, iPhone), I found the Python and BoostPython is big in size. I’m also not satisfy with Qt Framework, it’s big and the code is quite complex. Since the goal of the engine is become small and fast, I questioned if C++ is fit for the project? I believe that C++ programmer have a trend of over design things.

In the meantime, I was learning Javascript/Html5 and Lua. I suddenly became a fan of functional-programming. I can’t stop asking myself — What if there is no classes in the core? What if the low level interfaces are global functoins? What if I only carry data by structures? … I believe if I rewrite the low-level in C, it will limit my ambition so it can prevent me from over design. In the high-level such as AI, Game Play, Editor and Scene Organization sometimes we need the concept of Object and Inheritance, we can use script language to fulfill the demands.

At the end, I decided to rewrite the whole low-level engine in C as known as the exSDK, and use Lua as my script language for doing the high level jobs.

First demo in exSDK

This is the first demo I build from the new exSDK. The demo shows me four important things:

  1. It is simple and clear to define low level engine through global functions and structures.
  2. Though you write the code in C, a custom Class structure is needed for carrying properties, serialization and inheritance.
  3. Function based low level architecture is easier in exposing interface to script(Lua) level.
  4. Coroutine and Closure are feasible if the script language supported.

You can download and run the Demo below. In Windows, clicked the batch file demo_test_coroutine.bat and demo_test_invoke.bat.
In Mac, run the sh file through Terminal $sh demo_test_coroutine.sh and $sh demo_test_invoke.sh. I only test them in Windows 7 and Mac OSX 10.7 – Lion.

You can find all the Lua scripts involved in the Demo in data/assets/scripts. If you are familiar with Unity, you will see how the demo works quickly.

More research on Unity3D

I need more experience in making 2D games by Entity-Component system. With my Unity skill growing, I see more good concepts in it. I started building a series of 2D tools in Unity which at the end turn to ex2D. I found the concept of ScriptableObject is great for custom assets. It also shows a way to write editor in scripts in the runtime.

 

Now I collected enough information. And I decide to develop my own script-driven GUI framework. The GUI will be renderred in OpenGL so both game and editor share the basic codes. There will be less native codes for GUI in this way, so cross-platform editor become easier. I also use SDL and PhysicsFS to set me free from other cross-platform work, especially from handling events and managing native windows.
