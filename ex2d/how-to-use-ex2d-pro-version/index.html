How To Use ex2D Pro Version

BUY ex2D Pro

How to use ex2D Pro version:

 

** Important ** Do not import the package directly into the project you’re working on, especially don’t import the ex2D Pro package into the project with ex2D Indie version installed.

 

Instead you need to create a test environment to do the necessary changes to ex2D source files and compile the source file to DLLs (just like the ones in ex2D Indie package), and copy the DLLs over to your real project.

 

Step 1: Import package to your test project.

 

  1. Extract the .unitypackage file to any folder.
  2. Create an empty Unity project, and double click the .unitypackage file from previous step, this will open a dialogue for importing the package into your current Unity project.
  3. If the dialogue doesn’t show, go to your Unity editor and go to Assets -> Import Package -> Custom Package. Then select the package file. The import dialogue should show up.
  4. Click “import” button and the contents of the package should show up in the Project Panel.

Step 2: Do your change to the ex2D source files in the project you just created. And do your test to make sure them work properly.

Step 3: Create a folder structure for compiling.

Create an empty folder, copy the project folder with source code into it, and rename the project folder to “ex2d_dev“.

Copy the “Makefile_34“, “Makefile_35” file into the ex2d_dev folder, you should have a file structure like this:

Container_folder

|-ex2d_dev/(this is the test project you’ve created in step 1)

   |- Assets

   |- Library

   |- Makefile

Then you should create a folder in the same level of ex2d_dev named ex2d_submit and create folder structure as below:

|-ex2d_submit/

   |- Assets

      |- ex2D

           |- Core

           |- Editor

 

The whole folder structure should look like this:

Step 4: Compile Files

For Mac users, Open your Terminal, and type:

 Make -f Makefile_34 rebuild_ex2d

for Unity3.4,

or type

 Make -f Makefile_35 rebuild_ex2d

for Unity3.5.

Which will build the ex2D.Runtime.dll and ex2D.Editor.dll into ex2d_submit/Assets/ex2D/Core and ex2d_submit/Assets/ex2D/Editor directories. You can copy the content from ex2d_submit/Assets/ex2D to your real project folder for the update.

Don’t miss the Shaders and Editor/Resource folders! You should copy them over together with DLL files, under the same structure!

For Windows users, you should install MinGW or cygwin, and make sure make.exe is installed. Then you should edit the Makefile, change the line:

UNITY_PATH = /Applications/Unity/Unity.app/Contents/Frameworks

to the proper Unity installed path and then build it.
