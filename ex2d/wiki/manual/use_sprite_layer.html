Use Sprite Layer

Theory

When you add a Sprite or SpriteFont component in a GameObject by default, you will find ex2D automatically add Layer2D component either. What is this component and what does it do ?

Layer2D is designed to let the user adjust the z order of the sprites in the scene. There are two properties in Layer2D â€“ Layer and Bias.

The max number of layer in ex2D is 32. And you can calculate the z by the formular: 

z = (Camera.main.farClipPlane - Camera.main.nearClipPlane) / maxLayer * ( layer + bias );

The Render Order

Layer is the main coefficient in calculate z order. The larger value of the layer, the more depth the sprite will be. In other word, lower layer will render as if it is infront of the screen.

Bias is a float value from 0.0 to 1.0. It affects the render order of the sprites with the same layer. The larger value means the sprites will go behind more.

Free Adjustment

When the layer component added in a GameObject, you can't not drag the z-handle in the scene to adjust the GameObject's z position.

If you would like to free control the GameObject/Sprite, remove the Layer2D component from it.
